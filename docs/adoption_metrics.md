# Adoption Metrics

This document tracks project traction over time using mostly GitHub-native signals.

The goal is not to claim a formal user count. The goal is to keep a lightweight record of growth, engagement, and early adoption signals for `napari-chat-assistant`.

## How To Use This

- update this file once per week, once per release, or whenever you want to record a meaningful snapshot
- use GitHub as the primary source of truth unless another source is explicitly listed
- treat these numbers as adoption signals, not as proof of exact active-user counts

## Main Signals To Track

| Signal | Why it matters | Source |
| --- | --- | --- |
| Stars | Broad interest and discoverability | GitHub repository page |
| Forks | Stronger technical interest and experimentation | GitHub repository page |
| Watchers | Ongoing interest in project updates | GitHub repository page |
| Views (14d) | Discovery and traffic | GitHub Insights -> Traffic |
| Clones (14d) | Stronger intent than views | GitHub Insights -> Traffic |
| Unique contributors | Community formation | GitHub contributors page |
| Issues opened/closed | Real usage and feedback loop | GitHub Issues |
| Releases | Project maturity and shipping cadence | GitHub Releases |
| Release downloads | Evidence of release consumption if available | GitHub Releases |
| Citations | Academic uptake | GitHub citation box, DOI services, papers |
| Mentions | External visibility | blog posts, talks, courses, social posts |

## Where To Find Each Metric

### GitHub repository page

Use the repo header for:
- stars
- forks
- watchers

Use the repository tabs/pages for:
- contributors
- issues
- pull requests
- releases

### GitHub Insights

Use `Insights -> Traffic` for:
- views
- unique visitors if shown
- clones
- unique cloners if shown

### Releases

Use the Releases page for:
- total releases
- latest release
- release notes quality
- download counts if visible

### Citations and mentions

Use:
- GitHub citation support from `CITATION.cff`
- papers, talks, slides, blog posts, lab pages, or course material

## How To Talk About These Numbers

Good wording:

- "The project is showing encouraging early traction."
- "GitHub traffic shows continuing discovery and clone activity."
- "We are tracking adoption through GitHub traffic, repository engagement, releases, and community feedback."

Avoid wording like:

- "We have a large established user base" unless you have stronger evidence
- "Clones equal users"
- "Views prove adoption"

## What Counts As Stronger Evidence Later

Stronger evidence of adoption would include:
- repeat issues from different users
- multiple contributors
- release download activity
- citations in papers or slide decks
- community examples, tutorials, or workflow mentions
- users coming back across multiple releases
